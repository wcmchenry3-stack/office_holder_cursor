Table parsing

Which table and how to parse it.

When checked, one column holds both start and end; term end is grayed out and matches start.

Do not extract party; store null. Party column is grayed out — do not enter.

Use the column number above to read district from the table; set all rows to At-Large; or set all to No District and ignore the column.

Infer link, party, and term columns from the first row instead of using fixed column numbers.

Use when the table's link column is on the right (e.g. some senate tables).

Find date in infobox: get full dates from bios. Table has years only: table has year ranges only (stores start/end year). Choose one or not applicable.

When the table uses rowspan: carry forward name, party, and district when the name cell is empty; one result row per table row.

After parsing, merge consecutive terms for the same person when the gap is 1 day or less (e.g. same-day handoff). Use with Parse rowspan.

Use for US House tables: match infobox on the House of Representatives link.
